一、导航设置（Navigation Settings）

导航栏 logo 图像设置：
logo：未给出具体配置，可设置为一个图像路径以在导航栏显示 logo。
display_title：设置为true时，会显示导航栏标题。
fixed：设置为false，表示导航栏不固定在页面顶部。
菜单设置：
通过menu配置项，可以定义不同的页面链接和对应的图标。例如：
首页: / || fas fa-home表示首页链接为根路径，图标为 Font Awesome 的fas fa-home。
清单||fa fa-heartbeat:下可以定义子菜单，如音乐: /music/ || fas fa-music表示音乐页面的链接和图标。

二、代码块设置（Code Blocks Settings）

主题和样式设置：
theme：可以设置代码块的主题，如pale night等。
macStyle：设置为false，可能是不采用某种特定的 Mac 风格。
height_limit：设置为false，表示不限制代码块的高度。
word_wrap：设置为true，表示代码块中的文本可以自动换行。
工具栏设置：
copy：设置为true，表示显示复制代码的按钮。
language：设置为true，表示显示代码块的语言标识。
shrink：设置为false，表示不收缩代码块，也可以设置为true收缩代码块或none隐藏收缩按钮并展开代码块。
fullpage：设置为false，表示不启用全屏显示代码块的功能。

三、社交媒体链接设置（Social media links）

可以通过配置social项来添加社交媒体链接，格式为图标: 链接 || 描述 || 颜色，例如：
fab fa-github: https://github.com/xxxxx || Github || '#24292e’表示添加了一个指向 GitHub 的链接，图标为 Font Awesome 的 GitHub 图标，描述为Github，颜色为#24292e。

四、图像设置（Image Settings）

网站图标和头像设置：
favicon：设置网站的图标路径，如/img/favicon.png。
avatar：可以设置头像的图像路径和是否应用特效。
页面横幅图像设置：
disable_top_img：设置为false，表示不禁用所有页面的横幅图像。
default_top_img：如果某个页面没有设置特定的横幅图像，将显示默认的横幅图像。
可以为特定页面如首页、存档页、标签页、分类页等设置单独的横幅图像。
tag_per_img和category_per_img可以为每个标签和分类设置特定的横幅图像。
footer_img：设置为false，表示不设置页脚的背景图像。
文章封面设置：
cover：可以设置文章封面的显示选项，如在首页、侧边栏、存档页是否显示文章封面，以及默认封面的设置。
error_img：设置当图片加载失败时的替代图片路径。
error_404：可以配置简单的 404 页面，包括是否启用、副标题和背景图像。

五、首页设置（Index page settings）

首页顶部图像和站点信息设置：
index_site_info_top：设置首页站点信息的位置，如距离顶部的距离。
index_top_img_height：设置首页顶部图像的高度。
副标题设置：
subtitle：可以设置是否启用副标题，以及副标题的效果和来源等选项。例如，可以设置副标题的来源为第三方服务 API，如hitokoto.cn等，并可以设置副标题的效果为打字机效果。
首页文章布局设置：
index_layout：可以设置首页文章的布局方式，如封面在左信息在右、封面在右信息在左等多种布局方式。
首页文章内容显示设置：
index_post_content：可以设置在首页显示文章的介绍内容，包括显示方式（如描述、自动摘要等）和长度限制。

六、文章设置（Post Settings）

目录设置：
toc：可以设置文章是否显示目录，以及目录的样式和选项，如是否显示数字、是否展开等。
版权设置：
post_copyright：可以设置文章的版权信息，包括是否启用、作者链接、许可证等。
赞助设置：
reward：可以设置是否启用赞助功能，以及显示的文本和二维码链接等。
在线编辑设置：
post_edit：可以设置是否启用文章在线编辑功能，以及编辑链接的地址。
相关文章设置：
related_post：可以设置是否启用相关文章功能，以及显示的文章数量和排序方式。
文章分页设置：
post_pagination：可以设置文章的分页方式，如1表示 “下一篇” 链接到旧文章，2表示链接到新文章，false表示禁用分页。
文章过期通知设置：
noticeOutdate：可以设置是否启用文章过期通知，包括通知的样式、显示时间限制和位置等。

七、页脚设置（Footer Settings）

所有者信息设置：
footer：可以设置页脚的所有者信息，包括是否启用、起始年份等。
copyright：设置为true时，会显示主题和框架的版权信息。

八、侧边栏设置（Aside Settings）

侧边栏显示设置：
aside：可以设置侧边栏的显示选项，如是否启用、隐藏按钮、位置等。
display：可以设置在侧边栏显示的内容，如存档、标签、分类等。
card_author：可以设置作者卡片的显示选项，包括描述、按钮等。
card_announcement：可以设置公告卡片的显示内容。
card_recent_post：可以设置最近文章卡片的显示选项，如显示数量、排序方式等。
card_newest_comments：可以设置最新评论卡片的显示选项，如是否启用、显示数量、排序方式等。
card_categories：可以设置分类卡片的显示选项，如显示数量、是否展开等。
card_tags：可以设置标签卡片的显示选项，如显示数量、颜色、排序方式等。
card_archives：可以设置存档卡片的显示选项，如类型、格式、排序方式等。
card_post_series：可以设置文章系列卡片的显示选项。
card_webinfo：可以设置网站信息卡片的显示选项，如文章数量、最后更新日期等。

九、右下角按钮设置（Bottom right button）

按钮位置设置：
rightside_bottom：设置右下角按钮距离底部的距离。
语言转换设置：
translate：可以设置是否启用语言转换功能，包括按钮的文本、默认语言、转换延迟等选项。
阅读模式设置：
readmode：设置为true时，启用阅读模式。
黑暗模式设置：
darkmode：可以设置黑暗模式的显示选项，如是否启用、切换按钮、自动切换模式等。
按钮顺序设置：
rightside_item_order：可以设置右下角按钮的显示顺序。

十、全局设置（Global Settings）

锚点设置：
anchor：可以设置页面滚动时 URL 根据标题 ID 更新的选项，以及点击标题滚动并更新锚点的选项。
图片标题设置：
photofigcaption：设置为false，可能表示不启用图片标题功能。
复制设置：
copy：可以设置复制内容时的选项，如是否启用版权信息等。
字数统计设置：
wordcount：可以设置是否启用字数统计功能，包括文章字数统计、阅读时间统计、网站总字数统计等。
访问统计设置：
busuanzi：可以设置是否启用不蒜子统计功能，包括网站 UV、PV 和页面 PV 的统计。

十一、数学公式设置（Math）

数学公式引擎选择：
math：可以选择使用mathjax或katex作为数学公式引擎，也可以不使用数学公式引擎。
per_page：设置为true时，每个页面都会加载数学公式脚本。
MathJax 设置：
enableMenu：设置为true时，启用 MathJax 的上下文菜单。
tags：可以设置 MathJax 的编号方式。
KaTeX 设置：
copy_tex：设置为false，可能表示不启用复制 KaTeX 公式的功能。

十二、搜索设置（Search）

搜索引擎选择：
search：可以选择使用algolia_search、local_search或docsearch作为搜索引擎，也可以不使用搜索功能。
不同搜索引擎的设置：
algolia_search：可以设置 Algolia 搜索的结果数量等选项。
local_search：可以设置本地搜索的预加载、显示结果数量等选项。
docsearch：可以设置 Docsearch 的应用 ID、API 密钥、索引名称等选项。

十三、分享系统设置（Share System）

分享系统选择：
share：可以选择使用sharejs或addtoany作为分享系统，也可以不使用分享功能。
不同分享系统的设置：
sharejs：可以设置 Share.js 的分享站点。
addtoany：可以设置 AddToAny 的分享站点。

十四、评论系统设置（Comments System）

评论系统选择：
comments：可以选择使用多种评论系统，如Disqus、Disqusjs、Livere、Gitalk、Valine、Waline、Utterances、Facebook Comments、Twikoo、Giscus、Remark42、Artalk等，也可以不使用评论功能。
不同评论系统的设置：
对于每种评论系统，都有相应的配置项，如Disqus的shortname和apikey，Gitalk的client_id、client_secret、repo、owner和admin等。

十五、聊天服务设置（Chat Services）

聊天服务选择：
chat：可以选择使用chatra、tidio或crisp作为聊天服务，也可以不使用聊天功能。
不同聊天服务的设置：
对于每种聊天服务，都有相应的配置项，如chatra的id，tidio的public_key，crisp的website_id等。

十六、分析设置（Analysis）

可以配置多种分析工具，如百度统计、谷歌分析、Cloudflare 分析、微软 Clarity 和 Umami 分析等，通过设置相应的配置项来启用这些分析工具。

十七、广告设置（Advertisement）

Google Adsense 设置：
google_adsense：可以设置是否启用 Google Adsense 广告，以及自动广告、客户端等选项。
手动插入广告设置：
ad：可以在首页、侧边栏和文章中手动插入广告代码。

十八、验证设置（Verification）

可以配置网站验证信息，如谷歌网站验证、百度网站验证等。

十九、美化和效果设置（Beautify / Effect）

主题颜色设置：
theme_color：可以设置主题颜色，包括主色、分页器颜色、按钮悬停颜色等。
分类和标签页面 UI 设置：
category_ui和tag_ui：可以设置分类和标签页面的用户界面，选择与首页相同的 UI 或默认的存档页面 UI。
圆角设置：
rounded_corners_ui：设置为true时，UI 元素将显示圆角。
文本对齐设置：
text_align_justify：设置为false，表示不拉伸文本行以使每行宽度相等。
遮罩设置：
mask：可以设置在页眉和页脚添加遮罩。
加载动画设置：
preloader：可以设置加载动画的选项，如是否启用、动画类型和主题等。
页面过渡设置：
enter_transitions：设置为true时，启用页面过渡效果。
默认显示模式设置：
display_mode：可以设置默认显示模式为亮模式或暗模式。
文章内容美化设置：
beautify：可以设置是否启用文章内容美化功能，以及美化的字段、标题前缀图标和颜色等选项。
全局字体设置：
font：可以设置全局字体大小、代码字体大小、字体家族等选项。
博客标题字体设置：
blog_title_font：可以设置博客标题的字体链接和字体家族。
分割线图标设置：
hr_icon：可以设置分割线图标是否启用、图标和位置等选项。
激活能量模式设置：
activate_power_mode：可以设置是否启用激活能量模式，包括彩色效果、震动效果和是否在移动设备上启用等选项。
背景效果设置：
canvas_ribbon、canvas_fluttering_ribbon、canvas_nest、fireworks、click_heart和clickShowText等配置项可以设置不同的背景效果，如丝带效果、鼠标点击效果等。

二十、灯箱设置（Lightbox Settings）

可以选择使用fancybox或medium_zoom作为灯箱插件，也可以不使用灯箱功能。

二十一、标签插件设置（Tag Plugins settings）

系列设置：
series：可以设置文章系列的显示选项，如是否启用、排序方式等。
ABCJS 设置：
abcjs：可以设置是否启用 ABCJS 音乐符号插件。
Mermaid 设置：
mermaid：可以设置是否启用 Mermaid 图表插件，以及编写 Mermaid 图表的方式和主题等选项。
chartjs 设置：
chartjs：可以设置是否启用 chartjs 图表插件，以及图表的字体颜色、边框颜色等选项。
Note 设置：
note：可以设置 Note 标签的样式，如简单风格、现代风格、扁平风格等。

二十二、其他设置（Other Settings）

PJAX 设置：
pjax：可以设置是否启用 PJAX，以及排除某些页面不使用 PJAX。
APlayer 注入设置：
aplayerInject：可以设置是否启用 APlayer 注入功能，以及是否在每个页面都注入。
Snackbar 设置：
snackbar：可以设置是否启用 Snackbar 通知，以及通知的位置、背景颜色等选项。
Instant.page 设置：
instantpage：设置为false，表示不启用 Instant.page 功能。
Pangu 设置：
pangu：可以设置是否启用 Pangu 插件，以及在哪个字段（站点或文章）使用 Pangu。
Lazyload 设置：
lazyload：可以设置是否启用 Lazyload 插件，以及在哪个字段（站点或文章）使用 Lazyload。
PWA 设置：
pwa：可以设置是否启用渐进式 Web 应用（PWA）功能，以及相关的配置项。
Open Graph meta 标签设置：
Open_Graph_meta：可以设置是否启用 Open Graph meta 标签，以及相关的选项。
CSS 前缀设置：
css_prefix：设置为true时，会添加供应商前缀以确保兼容性。
注入设置：
inject：可以在页面的头部和底部注入 CSS 和脚本代码。
CDN 设置：
CDN：可以设置内部和第三方脚本的 CDN 提供商，以及版本号的添加和自定义格式等选项。
